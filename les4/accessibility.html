Accesibilidad

Internet está diseñado para trabajar con todas las personas, independientemente del hardware y software que usen, el idioma que hablen, o las capacidades que posean. Una página web accesible es una página web que cumple esta condición.

Para que una página web cumpla esta condición, es necesario que los desarrolladores de la misma hagan el esfuerzo de eliminar las barreras que esta naturalmente posee para ciertos usuarios.

Por qué crear páginas accesibles

La accesibilidad ayuda a la inclusión social para todas aquellas personas que, por cualquier motivo, tienen problemas para utilizar Internet de forma normal. Este grupo no se limita a personas con discapacidades, sino que también beneficia a personas que tengan una conexión de mala calidad, equipos limitados, etc.

La accesibilidad aumenta el rango de clientes al que puedes llegar al incluir a todas aquellas personas que se verían excluídas de un sitio web no accesible. En países desarrollados, la cantidad de personas con discapacidades reconocidas llega hasta el 25% por lo que es muy probable que este grupo represente una cantidad importante de potenciales consumidores de una empresa.

Las pautas de desarrollo de web accesibles se entrelazan con otras pautas recomendadas y buenas prácticas. Muchas de las pautas a seguir para construir una página web más accesible son, en realidad, las pautas más aconsejables para elaborar código HTML, CSS y JS de calidad.

Cómo crear páginas accesibles

Los principios básicos del desarrollo accesible se enfocan a eliminar las barreras comunmente encontradas por las personas discapacitadas o con herramientas de mala calidad. Algunos de estos principios son:

Provee texto alternativo al contenido multimedia: Las imágenes de una página web deberían utilizar el atributo "alt" para explicar qué representan dichas imágenes. Los elementos de audio y vídeo deben tener subtítulos y/o transcripciones del contenido. Estas pautas ayudan no solo a las personas discapacitadas, sino también a usuarios con mala conexión, que no entiendan bien el idioma, o que se encuentren en un lugar donde no puedan escuchar o visualizar contenido de forma clara.

Crea documentos con una estructura lógica: Un documento bien estructurado no es solo más facil de entender y editar para el desarrollador, sino que también es más fácilmente interpretado por programas auxiliares (e.g. lectores de pantalla).

Permite a los usuarios saltar elementos repetitivos: Los elementos que se repiten en diversas páginas de un sitio web, como el encabezado o el menú de navegación, deberían ser fáciles de ignorar para los usuarios que utilizan lectores de pantalla.

No utilices colores para mostrar información: Los colores pueden ser una herramienta para hacer una página más simple y fácil de entender, pero no deben, por si solos, representar información. En caso contrario, las personas daltónicas y aquellas que necesiten lectores de pantalla no podrán entender dicha información.

Escribe contenido de forma clara y concisa: No utilices términos extraños, utiliza fuentes fácilmente legibles, y no "contamines" el contenido con secciones no relacionadas.

Ajústate a los estándares: Una página que se ajuste a los estándares de HTML y CSS será, de base, más accesible que una página que los ignore, ya que las herramientas de accesibilidad como lectores de texto intentan utilizar estos estándares para realizar su trabajo. Estos estándares además han sido desarrollados con el objetivo de no dificultar la accesibilidad. Incluso sin el objetivo de accesibilidad en mente, ajustarse a los estándares sigue siendo beneficioso, ya que estos ayudan al desarrollador a producir un código más fácilmente mantenible y de mayor calidad.
